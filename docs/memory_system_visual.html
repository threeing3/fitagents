长期记忆系统架构图

这张图把项目里的长期记忆系统拆成四层：写入层、存储层、检索层、使用层。记住一个核心：长期记忆不是聊天记录仓库，而是经过校验、分类、向量化和压缩的用户状态系统。

写入和校验 存储和压缩 检索和重排 上下文使用

1. 写入层

从用户消息里提取候选记忆，再由 MemoryVerifier 过滤错误、临时、冲突信息。

2. 存储层

LongTermMemory 保存原子记忆，MemoryCatalog 做目录，MemoryBlock 做压缩摘要。

3. 检索层

向量、BM25、实体、重要性、近期性多路召回，再统一 rerank。

4. 使用层

ContextBuilder 把记忆、风险、知识、规则和模板组装成 Agent 本轮上下文。

主流程：从一句话到长期记忆，再到下一轮个性化回复

横向读：每一行是一层；纵向看：每列是同一阶段发生的事。

写入层

用户消息

例如：最近膝盖疼，但还想继续练腿。

profile.extract

抽取档案字段、纠错信息和候选长期记忆。

MemoryVerifier

拒绝错误记忆，降级临时状态，处理纠错冲突。

memory.write

只有通过校验的候选才真正写库。

MemoryManager

生成摘要、实体、embedding，并更新目录和语义块。

存储层

LongTermMemory

原子记忆：content、summary、category、importance、confidence。

entities

动作、症状、疾病、药物、饮食、恢复、目标。

evidence

记录来源表和 id，让 observation/opinion 有证据。

MemoryCatalog

按 risk、training、nutrition 等类别生成目录摘要。

MemoryBlock

把 profile、risk、preference 压缩成固定预算摘要块。

检索层

新问题

例如：今天腿怎么练？

过滤

按 user_id、status、category、memory_network、fact_kind、时间过滤。

多路召回

semantic、BM25、entity、temporal 同时召回候选。

去重

按 memory.id 合并候选，避免重复进入重排。

rerank

综合语义、关键词、实体、重要性、近期性和风险优先级。

使用层

ContextBuilder

本轮对话统一入口，决定要加载哪些上下文。

分组记忆

world、experience、observation、opinion 分开使用。

风险优先

active_risk_notes 和 decision_rules 优先于 opinion。

知识和模板

教练知识 RAG、决策规则、计划模板一起进入上下文。

Coach Agent

生成个性化回复或计划，并保存 trace。

核心表各自负责什么

表/结构作用
LongTermMemory具体记忆条目，保存内容、摘要、类别、实体、证据、向量、状态和有效期。
MemoryCatalog记忆目录，按类别汇总最近和重要记忆，用来快速了解某类记忆概况。
MemoryBlock压缩语义块，把高频上下文压成小摘要，比如 profile、risk、preference。
MemoryLink记忆关系，主要用于纠错、替代、更新和冲突关系。

四种 memory_network

网络下游如何使用
world用户事实，可以作为事实进入回答。
experience历史建议或 agent 行动，作为经验参考。
observation从日志和行为中总结出的模式，用作趋势参考。
opinion教练观点，必须显式标记，不能压过事实和规则。

写入时发生什么

候选提取从自然语言里抽出健康、训练、饮食、恢复、偏好等候选记忆。
校验防止把临时状态、否认信息、错误风险写成长期事实。
持久化生成 summary、entities、evidence、embedding 后写入数据库。

检索时为什么多路召回

向量找语义相近，比如“膝盖不舒服”和“膝关节疼痛”。
BM25找精确词，比如“赛治”“甲亢”“深蹲 100kg”。
实体优先召回膝盖、深蹲、睡眠、药物这类关键对象。
重要/近期避免漏掉高风险和最近状态。

最终怎么进入回答

ContextBuilder拿到相关记忆、压缩块、风险、知识和规则。
风险优先健康风险和结构化规则优先于教练观点。
Agent 回复模型基于整理后的上下文生成回答，代码再保存日志。

快速记忆：LongTermMemory 是具体记忆，MemoryCatalog 是目录，MemoryBlock 是压缩摘要，MemoryLink 是纠错关系；MemoryVerifier 负责防止乱写，search_memories 负责找回相关记忆，ContextBuilder 负责把记忆放进本轮 Agent 上下文。
